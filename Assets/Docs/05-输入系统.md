# 输入系统

> `InputManager` 基于 Unity Input System (新版)，处理键盘/鼠标输入。当前主要用于相机移动控制。

---

## 1. 为什么用新版 Input System

- 原生支持多种输入设备（键盘、鼠标、手柄、触屏）统一映射
- 事件驱动 + 轮询双模式
- 通过 `.inputactions` 资产可视化配置，减少硬编码
- 支持 Action Map 切换（Play / Dialogue / UI 不同场景用不同映射）

---

## 2. Input Action 资产

位置：`Assets/_Assets/InputAction/PlayerInputAction.inputactions`

自动生成的 C# 包装类：`Assets/_Assets/InputAction/PlayerInputAction.cs`

### Action Map 一览

| Map | Action | 类型 | 绑定 |
|-----|--------|------|------|
| **Play** | CameraMove | Value (Vector2) | WASD |
| **Play** | Camera Change | Value (1D Axis) | 滚轮上下 |
| **Dialogue** | Next Line | Button | Space / 鼠标左键 |
| **Dialogue** | Skip | Button | Esc 长按 1 秒 |
| **UI** | (未命名) | Button | 未绑定 |

### CameraMove 的 Composite Binding

使用 `2D Vector` 复合绑定：
- W → Up
- S → Down
- A → Left
- D → Right

### Camera Change 的 Composite Binding

使用 `1D Axis` 复合绑定：
- 滚轮上 → Positive
- 滚轮下 → Negative

## 3. InputManager 实现

```csharp
public class InputManager : ManagerBase<InputManager>
{
    [SerializeField] private PlayerInput playerInput;       // Unity 组件
    private PlayerInputAction playerInputAction;             // C# 包装类

    public Vector3 cameraMoveDirection = Vector3.zero;       // 已归一化的 3D 方向

    protected override void OnInit()
    {
        playerInput = GetComponent<PlayerInput>();
        playerInputAction = new PlayerInputAction();
    }

    protected override void OnTick(float deltaTime)
    {
        Vector2 raw = playerInputAction.Play.CameraMove.ReadValue<Vector2>();
        cameraMoveDirection = new Vector3(raw.x, 0, raw.y).normalized;
    }
}
```

### 数据流

```
按键 (WASD)
  → Input System 底层驱动
    → PlayerInputAction.Play.CameraMove.ReadValue<Vector2>()
      → InputManager.OnTick() 中每帧读取
        → 转为 Vector3(x, 0, y) 并归一化
          → cameraMoveDirection (public, 供 CameraManager 使用)
```

### 为什么每帧轮询而非事件驱动？

相机移动是连续操作，需要每帧获取最新的按键状态。事件驱动（`started/performed/canceled`）更适合离散操作（跳跃、攻击）。

### 获取相机移动方向

```csharp

// moveDir 已归一化，直接用
```

## 5. 场景配置

1. 在挂载 InputManager 的 GameObject 上添加 `Player Input` 组件
2. 将 `PlayerInputAction` 资产拖入 `Actions` 字段
3. `Default Scheme` 留空（自动匹配设备）
4. Behavior 选择 `Invoke Unity Events` 或 `Send Messages`（当前使用 C# 类直接访问，不依赖 Unity Events）
