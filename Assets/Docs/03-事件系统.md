# 事件系统

> `EventManager` 是一个基于字符串 + `Action<object>` 的全局事件总线，用于解耦各系统间的通信。

---

## 1. 设计动机

当系统 A 需要通知系统 B 某件事发生时，传统做法是 A 持有 B 的引用并直接调用。这在复杂项目中会导致：
- 网状依赖
- 难以追踪调用链
- 新增监听者需要修改发布者

事件系统将发布者与订阅者解耦：发布者只负责「喊出事件名」，不关心谁在听。

---

## 2. 核心实现

```csharp
public class EventManager : ManagerBase<EventManager>
{
    private readonly Dictionary<string, Action<object>> events = new();

    // 添加监听
    public void AddListener(string eventName, Action<object> callback)
    {
        if (!events.ContainsKey(eventName))
            events[eventName] = null;
        events[eventName] += callback;
    }

    // 移除监听
    public void RemoveListener(string eventName, Action<object> callback)
    {
        if (events.ContainsKey(eventName))
            events[eventName] -= callback;
    }

    // 发布事件
    public void Dispatch(string eventName, object data = null)
    {
        if (events.TryGetValue(eventName, out Action<object> callback))
            callback?.Invoke(data);
    }

    // 关闭时清空
    protected override void OnShutdown()
    {
        events.Clear();
    }
}
```

### 数据结构

`Dictionary<string, Action<object>>`：事件名 → 回调委托链。

`Action<object>` 本质是函数指针，支持 `+=` / `-=` 组合多个回调。同一事件名可注册多个监听者。

## 3. 使用方式

### 发布事件

在任意位置直接调用：

```csharp
// 无参数
EventManager.Instance.Dispatch("PlayerDead");

// 携带数据
EventManager.Instance.Dispatch("ScoreChanged", 100);
EventManager.Instance.Dispatch("CardPlayed", cardData);
```

### 订阅事件

```csharp
private void OnEnable()
{
    EventManager.Instance.AddListener("PlayerDead", OnPlayerDead);
    EventManager.Instance.AddListener("ScoreChanged", OnScoreChanged);
}

private void OnDisable()
{
    // 必须手动移除，否则对象销毁后回调仍存在（空引用异常）
    if (EventManager.Instance != null)
    {
        EventManager.Instance.RemoveListener("PlayerDead", OnPlayerDead);
        EventManager.Instance.RemoveListener("ScoreChanged", OnScoreChanged);
    }
}

private void OnPlayerDead(object data) { ... }
private void OnScoreChanged(object data)
{
    int score = (int)data;
}
```

## 4. 最佳实践

### 4.1 事件名管理

事件名应使用常量，避免手写字符串拼错：

```csharp
// 建议在 Utils 下新建 EventName.cs
public static class EventName
{
    public const string GameStart       = "GameStart";
    public const string PlayerTurnStart = "PlayerTurnStart";
    public const string CardPlayed      = "CardPlayed";
    public const string UnitMoved       = "UnitMoved";
    public const string LevelComplete   = "LevelComplete";
}

// 使用
EventManager.Instance.Dispatch(EventName.CardPlayed, card);
EventManager.Instance.AddListener(EventName.UnitMoved, OnUnitMoved);
```

### 4.2 生命周期管理

订阅者必须在销毁前移除监听，推荐在 `OnEnable` / `OnDisable` 中成对操作。

**为什么需要判空 `EventManager.Instance != null`？**
- 游戏退出时 Manager 的销毁顺序不确定，`OnDisable` 可能在 EventManager 被销毁后调用
- `Instance` 为 null 时 `RemoveListener` 会 NRE

### 4.3 数据传递

`object data` 可传任意类型，接收方需显式转型：

```csharp
private void OnCardPlayed(object data)
{
    CardData card = data as CardData;
    if (card == null) return;
    // ...
}
```

对于简单类型（int、string 等），直接用值类型转型即可，但注意装箱/拆箱开销。在非热点路径上可忽略。

### 4.4 避免循环事件

A 订阅事件 X，在回调中 Dispatch 事件 Y；B 订阅事件 Y，在回调中 Dispatch 事件 X。会导致无限递归栈溢出。

**解决方案**：设计事件流时保持单向，或使用标记位防止重入。
