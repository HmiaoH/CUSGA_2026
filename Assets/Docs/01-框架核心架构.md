# 框架核心架构：单例 + MOFM

> **MOFM = Manager of Managers**。所有全局管理器统一由 `GameEntry` 注册到 `ManagerCenter`，再统一驱动生命周期。

---

## 1. 设计目标

- **解耦**：各 Manager 之间不直接依赖，通过 EventManager 通信
- **统一生命周期**：所有 Manager 遵循 `Init → Tick → Shutdown` 三阶段
- **单例访问**：任意位置可通过 `XxxManager.Instance` 获取
- **可扩展**：新增 Manager 只需三步（写类、在 GameEntry 中声明字段、注册）

---

## 2. 层级架构

```
GameEntry (入口，挂载在场景 GameRoot 上)
  └── ManagerCenter (纯 C# 类，不继承 MonoBehaviour)
        ├── EventManager      (事件系统)
        ├── AudioManager      (音频)
        ├── UIManager         (UI)
        ├── ResourceManager   (资源)
        ├── SaveManager       (存档)
        ├── InputManager      (输入)
        ├── CameraManager     (相机)
        ├── CutsceneManager   (过场动画/Timeline)
        ├── DialogueManager   (对话)
        └── GameFlowManager   (游戏流程状态机)
```

## 4. 核心类详解

### 4.1 SingletonMono\<T\>

泛型 MonoBehaviour 单例基类。所有 Manager 通过它获得 `Instance` 静态属性。

```csharp
public abstract class SingletonMono<T> : MonoBehaviour where T : MonoBehaviour
{
    public static T Instance { get; private set; }

    protected virtual void Awake()
    {
        if (Instance != null && Instance != this)
        {
            Destroy(gameObject);   // 防止重复实例
            return;
        }
        Instance = this as T;
    }

    protected virtual void OnDestroy()
    {
        if (Instance == this)
            Instance = null;
    }
}
```

**关键行为**：
- `Awake` 中若已存在实例则销毁自身（保证全局唯一）
- `OnDestroy` 中清空 Instance 引用
- 所有子类 `Awake` 必须调用 `base.Awake()`

---

### 4.2 IManager 接口

定义 Manager 的三个生命周期阶段：

```csharp
public interface IManager
{
    void Init();              // 初始化（仅一次）
    void Tick(float deltaTime);  // 每帧更新
    void Shutdown();          // 退出清理
}
```

---

### 4.3 ManagerBase\<T\>

继承 `SingletonMono<T>` 并实现 `IManager`。核心设计是**状态保护 + 虚方法模式**：

- 公开的 `Init/Tick/Shutdown` 来自 IManager 的显式实现，负责检查 `initialized` 标志
- 实际逻辑写在 `protected virtual OnInit/OnTick/OnShutdown` 中，子类按需覆写

```csharp
public abstract class ManagerBase<T> : SingletonMono<T>, IManager where T : MonoBehaviour
{
    public bool initialized { get; private set; }

    void IManager.Init()
    {
        if (initialized) return;
        OnInit();
        initialized = true;
    }

    void IManager.Tick(float deltaTime)
    {
        if (!initialized) return;
        OnTick(deltaTime);
    }

    void IManager.Shutdown()
    {
        if (!initialized) return;
        OnShutdown();
        initialized = false;
    }

    protected virtual void OnInit() { }
    protected virtual void OnTick(float deltaTime) { }
    protected virtual void OnShutdown() { }
}
```

**为什么不把 Init/Tick/Shutdown 直接改成 public virtual？**
- 状态保护（防止重复 Init、防止未初始化时 Tick）统一在基类处理，子类无需关心
- `initialized` 字段封装，子类无法绕过

---

### 4.4 ManagerCenter

纯 C# 类，持有 `List<IManager>` 并统一调度：

```csharp
public class ManagerCenter
{
    private List<IManager> managers = new List<IManager>();

    public void Register(IManager manager) { ... }
    public void InitAll()     { foreach (var m in managers) m.Init(); }
    public void TickAll(float dt) { foreach (var m in managers) m.Tick(dt); }
    public void ShutdownAll() { /* 倒序关闭 */ }
}
```

**ShutdownAll 为什么倒序遍历？**
- Manager 之间可能存在依赖关系（如先注册的依赖后注册的），倒序关闭更安全
- 但在当前架构中 Manager 不应直接依赖，此处为防御性写法

---

### 4.5 GameEntry

整个框架的入口，挂载在场景中的 `GameRoot` 空物体上。职责：

1. **Awake**：`DontDestroyOnLoad` 防止切场景时被销毁；创建 ManagerCenter 并注册全部 Manager
2. **Update**：每帧调用 `managerCenter.TickAll(Time.deltaTime)`
3. **OnApplicationQuit**：调用 `managerCenter.ShutdownAll()`

```csharp
public class GameEntry : SingletonMono<GameEntry>
{
    [Header("Managers")]
    [SerializeField] private EventManager eventManager;
    [SerializeField] private ResourceManager resourceManager;
    [SerializeField] private AudioManager audioManager;
    [SerializeField] private UIManager uiManager;
    [SerializeField] private SaveManager saveManager;
    [SerializeField] private InputManager inputManager;
    [SerializeField] private CameraManager cameraManager;
    [SerializeField] private GameFlowManager gameFlowManager;
    [SerializeField] private CutsceneManager cutsceneManager;
    [SerializeField] private DialogueManager dialogueManager;

    private ManagerCenter managerCenter;

    protected override void Awake()
    {
        base.Awake();
        DontDestroyOnLoad(transform.root.gameObject);
        managerCenter = new ManagerCenter();
        RegisterManagers();
        managerCenter.InitAll();
    }

    private void Update()
    {
        managerCenter.TickAll(Time.deltaTime);
    }

    private void OnApplicationQuit()
    {
        managerCenter.ShutdownAll();
    }

    private void RegisterManagers()
    {
        // 按依赖顺序注册（先注册的被先 Init）
        managerCenter.Register(eventManager);
        managerCenter.Register(resourceManager);
        managerCenter.Register(audioManager);
        managerCenter.Register(uiManager);
        managerCenter.Register(saveManager);
        managerCenter.Register(inputManager);
        managerCenter.Register(cameraManager);
        managerCenter.Register(cutsceneManager);
        managerCenter.Register(dialogueManager);
        managerCenter.Register(gameFlowManager);  // 最后注册（依赖其他 Manager）
    }
}
```

**注册顺序说明**：
- GameFlowManager 最后注册，因为它在 `OnInit` 中会立即调用 `ChangeState`，可能触发其他 Manager 的接口
- 基础服务（Event、Resource、Audio）优先注册

## 5. 如何新增一个 Manager

1. 在 `Assets/Scripts/Managers/` 下新建脚本，继承 `ManagerBase<YourManager>`
2. 覆写 `OnInit/OnTick/OnShutdown` 实现具体逻辑
3. 在 `GameEntry` 中添加 `[SerializeField] private YourManager yourManager;` 字段
4. 在 `RegisterManagers()` 中添加 `managerCenter.Register(yourManager);`
5. 在场景中将 Manager 脚本挂载到 GameObject 上，并拖入 GameEntry 的对应字段

## 6. 与旧设计文档的差异

旧文档（`项目管理框架 单例 + MOFM.md`）为设计阶段的规划，实际实现有以下调整：

| 项目 | 设计文档 | 实际代码 |
|------|---------|---------|
| 接口可见性 | `IManager` 公开实现 | 显式接口实现（`void IManager.Init()`） |
| IsInitialized 命名 | `IsInitialized` | `initialized` |
| 命名空间 | 无 | Frameworks / Managers / Systems / Utils |
| 新增 Manager | 无 CameraManager | 有 CameraManager、InputManager（新版输入系统） |
| GameState 枚举 | 简单流程状态 | 细化到回合级别（PlayerTurnStart、WaitingForCard 等） |
