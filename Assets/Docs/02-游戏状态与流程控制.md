# 游戏状态与流程控制

> `GameFlowManager` 是整个游戏的心脏。它维护一个有限状态机 (FSM)，所有游戏进程的切换都通过它实现。

---

## 1. GameState 枚举

定义在 `Frameworks/GameState.cs`，共有 10 个状态：

```csharp
public enum GameState
{
    None,                // 初始/未定义
    OpeningCutscene,     // 开场背景动画
    PlayerTurnStart,     // 玩家回合开始
    WaitingForCard,      // 等待玩家选择/打出卡牌
    ResolvingCard,       // 正在结算卡牌效果（棋子移动）
    EnemyTurn,           // 敌方行动
    LevelComplete,       // 关卡完成
    TableInteraction,    // 与桌面物体互动
    Dialogue,            // 剧情对话
    GameOver             // 游戏结束
}
```

## 2. GameFlowManager 实现

```csharp
public class GameFlowManager : ManagerBase<GameFlowManager>
{
    public GameState CurrentState { get; private set; } = GameState.None;

    protected override void OnInit()
    {
        ChangeState(GameState.OpeningCutscene);
    }
}
```

### 核心方法：ChangeState

每次状态切换遵循 **Exit → Set → Enter** 三步：

```csharp
public void ChangeState(GameState newState)
{
    if (CurrentState == newState) return;

    ExitState(CurrentState);   // 1. 退出旧状态（释放资源、关闭 UI）
    CurrentState = newState;   // 2. 设置新状态
    EnterState(CurrentState);  // 3. 进入新状态（播放动画、打开 UI、触发事件）
}
```

### ExitState

根据退出的状态执行清理操作。当前大部分状态的 Exit 逻辑待实现：

```csharp
private void ExitState(GameState state)
{
    switch (state)
    {
        case GameState.OpeningCutscene:   break;
        case GameState.PlayerTurnStart:   break;
        case GameState.WaitingForCard:    break;
        case GameState.ResolvingCard:     break;
        case GameState.EnemyTurn:         break;
        case GameState.TableInteraction:  break;
        case GameState.LevelComplete:     break;
        case GameState.Dialogue:          break;
    }
}
```

### EnterState

根据进入的状态执行初始化操作。目前仅实现了 `EnterOpeningCutscene`：

```csharp
private void EnterState(GameState state)
{
    switch (state)
    {
        case GameState.OpeningCutscene:
            EnterOpeningCutscene();
            break;
        // 其余状态待实现
    }
}

private void EnterOpeningCutscene()
{
```

## 4. 与其他系统的协作

### 4.1 与 CutsceneManager 的协作

开场动画播放完毕后，CutsceneManager 通过 `onComplete` 回调通知。当前代码中 `EnterOpeningCutscene` 的回调参数为 `null`，后续应改为：

```csharp
private void EnterOpeningCutscene()
{
    CutsceneManager.Instance.PlayCutscene(CutsceneName.OPENING, () =>
    {
        ChangeState(GameState.PlayerTurnStart);
    });
}
```

### 4.2 与 EventManager 的协作

状态进入时可广播事件，供其他系统响应：

```csharp
// 示例：进入 Playing 相关状态时
EventManager.Instance.Dispatch("PlayerTurnStart");
EventManager.Instance.Dispatch("GameStart");
```

### 4.3 与 UIManager 的协作

状态切换需控制 UI 面板的开关：

```csharp
// Enter
UIManager.Instance.OpenPanel("HUDPanel");
UIManager.Instance.OpenPanel("DialoguePanel");

// Exit
UIManager.Instance.ClosePanel("HUDPanel");
UIManager.Instance.ClosePanel("DialoguePanel");
```

---

## 5. 设计原则

1. **单向依赖**：GameFlowManager 可以调用其他 Manager，反之不行。其他系统通过 `GameFlowManager.Instance.ChangeState()` 请求状态切换
2. **状态原子性**：每次 `ChangeState` 必须完整执行 Exit + Enter，不允许中间态
3. **外部触发**：卡牌打出、动画结束等事件触发状态切换，而非在 GameFlowManager 内部轮询
